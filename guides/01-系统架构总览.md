# 01 - 系统架构总览

> 理解设计理念，掌握核心架构

---

## 一、设计理念

### 1.1 为什么需要多Agent？

单个AI助手（ChatGPT/Claude）能对话，但无法：

- **同时追踪多个任务** — 你让它写文章，就忘了还在做研究
- **专业化分工** — 情报采集、深度研究、内容创作需要不同的"人设"
- **定时自动化** — 每天自动抓热点、生成日报
- **团队协作** — 情报总监采集 → 研究员分析 → 创作者产出

**多Agent系统 = 一个AI团队，各司其职，协同工作。**

### 1.2 核心设计原则

| 原则 | 说明 |
|------|------|
| **单一职责** | 每个Agent只负责一个领域 |
| **独立上下文** | Agent之间上下文隔离，互不干扰 |
| **总控调度** | 一个总控Agent负责路由和编排 |
| **极简通信** | Agent间通过简报格式通信，不传冗余信息 |
| **渐进部署** | 先核心Agent，再按需扩展 |

## 二、系统架构

### 2.1 整体架构图

```
┌─────────────────────────────────────────────┐
│                  用户 (Telegram)              │
└──────────────┬──────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────────┐
│            OpenClaw Gateway                  │
│         (单一进程，管理所有Agent)              │
└──────┬──────────┬──────────┬────────────────┘
       │          │          │
       ▼          ▼          ▼
   ┌───────┐  ┌───────┐  ┌──────────┐
   │CEO  │  │小秘 │  │专业Agents │
   │总控管家│  │聊天伙伴│  │(可多个)   │
   └───┬───┘  └───────┘  └────┬─────┘
       │                       │
       │  sessions_spawn       │
       ├──────────────────────►│
       │  派发任务              │
       │                       │
       │  sessions_send ◄──────┤
       │  汇报简报              │
       └───────────────────────┘
```

### 2.2 双主Agent架构

| Agent | ID | 角色 | 说明 |
|-------|----|------|------|
| **CEO** | `ceo` | 总控管家 | 路由、编排、定时任务、Fallback |
| **小秘** | `secretary` | 聊天伙伴 | 日常聊天、陪伴交流 |

- **CEO**是"老板"：接收任务，分发给专业Agent
- **小秘**是"朋友"：日常闲聊，不参与任务编排
- 用户可以分别通过不同的Telegram Bot与它们对话

### 2.3 专业Agent团队

| ID | 名称 | 角色 | 触发场景 |
|----|------|------|----------|
| `intel-director` | 情报总监 | 热点情报、选题推荐 | "今天有什么热点"、"帮我找选题" |
| `wechat-manager` | 公众号主管 | 公众号创作、选题策划 | "写一篇公众号"、"公众号选题" |
| `redbook-manager` | 小红书主管 | 小红书笔记、爆款分析 | "写小红书笔记"、"分析爆款" |
| `ops-director` | 运营总监 | 增长策略、数据分析 | "怎么涨粉"、"分析数据" |
| `product-manager` | 产品经理 | 需求分析、竞品分析 | "分析需求"、"看看竞品" |
| `tech-lead` | 技术总监 | 代码开发、架构设计 | "写代码"、"技术方案" |
| `finance-buddy` | 财务管家 | 收支管理、成本分析 | "本月收支"、"帮我记账" |
| `investor` | 投资专家 | 投资分析、市场研判 | "分析股票"、"投资建议" |
| `biz-advisor` | 商业顾问 | 商业模式、定价策略 | "这个项目值不值得做" |
| `knowledge-lib` | 知识管家 | 知识归档、笔记整理 | "帮我整理笔记"、"之前说的XX" |

## 三、核心机制

### 3.1 智能路由

用户发出指令 → CEO分析意图 → 路由到对应Agent

```
"帮我抓热点"        → intel-director (情报总监)
"写一篇公众号"      → wechat-manager (公众号主管)
"发个小红书"        → redbook-manager (小红书主管)
"怎么涨粉"          → ops-director (运营总监)
"分析一下需求"      → product-manager (产品经理)
"帮我写代码"        → tech-lead (技术总监)
"本月收支"          → finance-buddy (财务管家)
"分析一下股票"      → investor (投资专家)
"这个项目值不值"    → biz-advisor (商业顾问)
"帮我整理笔记"      → knowledge-lib (知识管家)
"今天有什么安排"    → secretary (小秘)
"其他"              → CEO 直接处理
```

### 3.2 Agent间通信

```
CEO → 专业Agent:  sessions_spawn (派发任务)
专业Agent → CEO:  sessions_send (汇报简报)
```

**极简简报格式**（专业Agent完成任务后）：
```
【极简简报】
✅ 任务: [任务名称]
📄 产出: [文件路径]
📊 统计: [关键数据]
💡 亮点: [一句话核心成果]
```

### 3.3 独立工作区

每个Agent拥有独立工作区，互不干扰：

```
~/.openclaw/
├── workspace/              # 小秘 工作区
├── workspace-ceo/      # CEO工作区
├── workspace-intel-director/      # 情报总监工作区
├── workspace-creator/      # 创作者工作区
└── ...
```

### 3.4 定时任务 (Heartbeat)

每个Agent可配置独立的心跳周期：

```json
{
  "heartbeat": {
    "every": "30m",
    "activeHours": {
      "start": "05:00",
      "end": "22:00",
      "timezone": "Asia/Shanghai"
    }
  }
}
```

- 情报总监：每30分钟检查一次（覆盖早间热点）
- 总控管家：每2小时检查一次
- 其他Agent：按需配置

## 四、关键技术概念

### 4.1 Gateway

- **是什么**: OpenClaw的核心守护进程
- **做什么**: 管理所有Agent的生命周期、处理消息收发
- **端口**: 默认 `18790`

### 4.2 Session

- 每个Agent的对话是一个Session
- Session Key格式: `agent:<agentId>:main`
- Agent间通信通过Session Key定位目标

### 4.3 SOUL.md

- Agent的"灵魂文件" — 定义身份、职责、沟通风格
- 类似于System Prompt，但以文件形式持久化
- 每个Agent都有独立的SOUL.md

### 4.4 Workspace

- Agent的"家目录" — 存放配置、记忆、产出文件
- 包含: SOUL.md, USER.md, AGENTS.md, MEMORY.md, TOOLS.md

## 五、通信拓扑

```
                    ┌──────────┐
                    │  用户     │
                    └────┬─────┘
                         │
              ┌──────────┼──────────┐
              ▼                     ▼
        ┌──────────┐         ┌──────────┐
        │  CEO   │◄───────►│  小秘  │
        │ (总控)   │         │ (聊天)   │
        └────┬─────┘         └──────────┘
             │
    ┌────┬───┴───┬────┬────┬────┬────┬────┬────┐
    ▼    ▼       ▼    ▼    ▼    ▼    ▼    ▼    ▼
 ┌────┐┌────┐┌────┐┌────┐┌────┐┌────┐┌────┐┌────┐
 │情报││公众号││小红书││运营││产品││技术││财务││投资│
 │总监││主管││主管││总监││经理││总监││管家││专家│
 └────┘└────┘└────┘└────┘└────┘└────┘└────┘└────┘
       ┌────┐┌────┐
       │商业││知识│
       │顾问││管家│
       └────┘└────┘
```

- CEO是中心节点，所有专业Agent通过它与用户交互
- 小秘独立运行，不参与任务编排
- 专业Agent之间不直接通信，通过CEO协调
