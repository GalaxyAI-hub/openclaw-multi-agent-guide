# 02 - 零基础上手指南

> 从0到1搭建你的多Agent系统

---

## 一、环境准备

### 1.1 系统要求

| 项目 | 最低要求 | 推荐配置 |
|------|----------|----------|
| 操作系统 | macOS 13+ / Ubuntu 20+ / Windows WSL2 | macOS 14+ (Apple Silicon) |
| Node.js | v20+ | v22 LTS |
| 内存 | 4GB | 8GB+ |
| 网络 | 需要访问LLM API和Telegram | 稳定网络 |

### 1.2 前置账号

1. **LLM API密钥** — 至少一个模型提供商的API Key
   - 智谱AI (GLM): [open.bigmodel.cn](https://open.bigmodel.cn)
   - OpenAI: [platform.openai.com](https://platform.openai.com)
   - OpenRouter: [openrouter.ai](https://openrouter.ai) (聚合多模型)
   
2. **Telegram账号** — 用于创建Bot并与Agent交互
   - 下载Telegram客户端: [telegram.org](https://telegram.org)

### 1.3 安装Node.js

```bash
# 推荐: 使用 nvm 安装
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
source ~/.zshrc  # 或 source ~/.bashrc
nvm install 22
nvm use 22

# 验证
node --version  # 应显示 v22.x.x
npm --version
```

## 二、安装OpenClaw

### 2.1 全局安装

```bash
npm install -g openclaw
```

### 2.2 验证安装

```bash
openclaw --version
openclaw doctor   # 检查环境是否就绪
```

### 2.3 初始化配置

```bash
openclaw init
```

这会在 `~/.openclaw/` 创建基础配置目录结构：

```
~/.openclaw/
├── openclaw.json     # 主配置文件
├── agents/           # Agent定义目录
├── workspace/        # 默认工作区
└── memory/           # 记忆存储
```

## 三、配置模型

### 3.1 在 openclaw.json 中配置模型

编辑 `~/.openclaw/openclaw.json`，添加模型提供商：

```json
{
  "models": {
    "mode": "replace",
    "providers": {
      "myProvider": {
        "baseUrl": "https://api.example.com/v1",
        "apiKey": "你的API密钥",
        "api": "openai-completions",
        "models": [
          {
            "id": "model-name",
            "name": "显示名称",
            "reasoning": false,
            "input": ["text"],
            "contextWindow": 128000,
            "maxTokens": 8192
          }
        ]
      }
    }
  }
}
```

### 3.2 常用模型提供商配置示例

**智谱AI (GLM)**:
```json
{
  "zai": {
    "baseUrl": "https://open.bigmodel.cn/api/coding/paas/v4",
    "apiKey": "你的智谱API Key",
    "api": "openai-completions",
    "models": [
      {
        "id": "glm-4",
        "name": "GLM-4",
        "reasoning": false,
        "input": ["text"],
        "contextWindow": 128000,
        "maxTokens": 8192
      }
    ]
  }
}
```

**OpenRouter (聚合多模型)**:
```json
{
  "openrouter": {
    "baseUrl": "https://openrouter.ai/api/v1",
    "apiKey": "你的OpenRouter API Key",
    "api": "openai-completions",
    "models": [
      {
        "id": "openai/gpt-4o",
        "name": "GPT-4o",
        "reasoning": false,
        "input": ["text", "image"],
        "contextWindow": 128000,
        "maxTokens": 16384
      }
    ]
  }
}
```

### 3.3 设置默认模型

```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "zai/glm-4",
        "fallbacks": ["openrouter/openai/gpt-4o"]
      }
    }
  }
}
```

模型ID格式: `<providerId>/<modelId>`

## 四、启用Agent间通信

这是多Agent系统的关键配置。编辑 `~/.openclaw/openclaw.json`:

```json
{
  "tools": {
    "sessions": {
      "visibility": "all"
    },
    "agentToAgent": {
      "enabled": true,
      "allow": ["ceo", "secretary", "intel-director"]
    }
  }
}
```

**配置说明**:

| 配置项 | 值 | 说明 |
|--------|----|------|
| `sessions.visibility` | `"all"` | 允许Agent间互相发现 |
| `agentToAgent.enabled` | `true` | 启用Agent间通信 |
| `agentToAgent.allow` | Agent ID数组 | 通信白名单 |

## 五、启动Gateway

### 5.1 启动

```bash

```

### 5.2 检查状态

```bash

```


## 七、常见问题

### Q: 安装时报权限错误？
```bash
# 不要用 sudo，用 nvm 管理的 npm
nvm use 22
npm install -g openclaw
```

### Q: Gateway启动失败？
```bash
# 检查端口是否被占用
lsof -i :18790
# 检查配置文件语法
python3 -m json.tool ~/.openclaw/openclaw.json
```

### Q: 模型调用失败？
- 检查API Key是否正确
- 检查网络能否访问API地址
- 检查模型ID格式是否正确 (`provider/model`)

### Q: Agent间无法通信？
- 确认 `tools.sessions.visibility` = `"all"`
- 确认 `tools.agentToAgent.enabled` = `true`
- 确认Agent ID在 `agentToAgent.allow` 列表中
- 重启Gateway使配置生效
